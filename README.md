## 切换机房的核心规则：免费但消耗流量

说说那个流量的坑。

机房迁移的本质是把你 VPS 上的整块磁盘镜像打包传输到新的数据中心。传多少东西，消耗多少流量——实测下来，每次迁移消耗的流量大约是 VPS 当前已用磁盘空间的 4 到 8 倍左右。

比如你的 20GB 硬盘用了 5GB，那迁一次大概要消耗 20GB 到 40GB 的月流量。一般套餐给 1TB 月流量，这个量级其实还好，但如果你买的是流量少的方案，或者本来当月就用了不少，频繁换机房就会加速把流量耗干。

另外有几点需要注意：

- **IP 会变**。换到新机房，你的公网 IPv4 地址会随之更换。如果你在这台 VPS 上跑了域名解析，切机房之后记得同步更新 DNS 记录。
- **切回原机房 IP 大概率不变**。比如从 DC6 换到 DC9，再换回 DC6，IP 通常还是原来那个，除非这期间原 IP 被别人占用了。
- **IP 被封的情况下，无法免费切换机房**。这是个很多人踩过的坑。如果你的 VPS IP 已经被封禁了，KiwiVM 里的 Migrate to another DC 功能会直接锁死，无法操作。这种情况只能花钱买新 IP，目前单独换 IP 的费用是 $7.39 美元。
- **迁移有冷却时间**。不是每次操作完马上又能再换。具体冷却周期因套餐和目标机房而异，一般是几周左右，以后台实际显示为准。

## 操作步骤：五步完成机房切换

整个过程用不了几分钟，不需要命令行，后台点几下就行。

1. 登录搬瓦工账户，进入 Services → My Services，找到你的 VPS 实例
2. 点击右侧的齿轮图标，选择 Open KiwiVM，进入控制面板
3. 在左侧菜单找到 **Migrate to another DC**，点击进入
4. 右侧会列出当前套餐支持的所有可选机房，选中目标机房，点击 **Confirm on next step**
5. 再次确认，点击 **Start Migration**，等待邮件通知迁移完成

整个迁移过程短则几分钟，长则几小时，具体取决于你 VPS 上存了多少数据。迁移进行中 VPS 依然可以访问，只有最后收尾阶段会有几秒到几分钟的短暂中断。

操作前建议先去 Snapshots 做一个快照备份，万一迁移过程出什么意外，数据还能恢复。

## 常见问题 FAQ

**Q：搬瓦工 VPS 切换机房收费吗？**
A：不收费。Migrate to another DC 是搬瓦工的免费自助功能，没有次数限制。但每次迁移会消耗月流量，消耗量约为当前已用磁盘大小的 4-8 倍，使用前注意流量余量。

**Q：切换机房之后我的数据会丢失吗？**
A：不会。迁移过程中磁盘数据完整保留，SSH 密码、配置文件、部署的应用全部跟着走，迁完之后只需要更新一下新 IP 就能继续用。

**Q：KVM 套餐可以切换到 CN2 GIA 机房吗？**
A：不可以。机房迁移只能在同一套餐允许的节点范围内进行，KVM 普通套餐无法迁移到 CN2 GIA 线路的机房。想用 CN2 GIA，要买 CN2 GIA-E 或更高端的套餐。

**Q：切换机房之后 IP 会变吗？**
A：会变。迁到新机房，IPv4 地址随之更换。如果你有绑定域名，记得更新 DNS 解析。如果从新机房切回原机房，原 IP 大概率还在（除非已被其他用户占用）。

**Q：哪个套餐的机房切换自由度最高？**
A：CN2 GIA-E（E-Commerce）套餐，支持 12 个以上机房互相迁移，包括洛杉矶 DC6/DC9、日本大阪软银、荷兰联通、加拿大温哥华、美国圣何塞等，是目前灵活度最高的常规套餐。
